My Docker Experience

Joshua Harris

Background

For about a year I've been developing almost exclusively on linux machines

  • I love and make use of unix tools/terminal
  • Made configurations to help my workflow
  • Love automation and writing scripts to perform repetitive actions

Problems?

In my work environment, many people use a windows machine

  • Made me think about how others would work on my project
  • My app required php/mysql, which can involve a bit of setup/configuration
  • Wanted to make that process as easy as possible
  • People who work/contribute to my project should just have to focus on the contribution, not setup

Cue Docker

What I knew about Docker was...

  • Docker used containers and could spin up an exact instance on any machine with Docker installed.
  • Docker Would allow me and others to focus on the product, not how to run it
    • Couple weeks back I attended a talk where someone demoed building a simple Docker container
    • This showed me what Docker was capable of

What is Docker

From Docker's Website

Docker allows you to package an application with all of its dependencies into a standardized unit for software development.

Docker Research

I started with figuring out what components would be needed for my basic lamp stack because Docker would require me to specify exactly what I needed

  • PHP
    • Composer
  • Apache Web Server
  • MySQL

Docker Hub

To get my Docker container up and running as fast as possible, I searched the Hub for containers which others put together and already offered what I needed.
