My Docker Experience

Joshua Harris

Background

I'm very comfortable developing in a linux environment due to all the tools/configurations I've set up and gotten use to. When starting my new intern position I was presented with a windows machine. This was no big deal because I was able to just use a virtual machine, putting myself in the environment which allows me to have a better workflow.

Problems?

The VM solution is great, being able to run any environment I want. But I ran into some annoyances.

  • Maintenence was annoying
  • Spending time troubleshooting issues was unproductive

Cue Docker

What I knew about Docker was...

  • Docker used containers and could spin up an exact instance on any machine with Docker installed.
  • Docker Would allow me to focus on the product, not how to run it
    • Couple weeks back I attended a talk where someone demoed building a simple Docker container
    • This showed me what Docker was capable of

Docker Research

I started with figuring out what components would be needed for my basci lamp stack

  • PHP
    • Composer
  • Apache Web Server
  • MySQL
