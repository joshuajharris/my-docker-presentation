My Docker Experience

Joshua Harris

Background

For about a year I've been developing almost exclusively on linux

  • I make use of unix tools
  • Have configurations to help my workflow
  • Prior to this experience I only had a basic understanding of what Docker could do
  • The computer given to me at work was a Windows machine
  • I spun up a virtual machine(VM) and after installation and setting up all my tools, was back to where I felt comfortable

Problems?

After using a VM for a couple months I ran into some annoyances

  • Had to focus some effort during work to maintenence
  • Spending time troubleshooting VM issues was unproductive
  • Performance inside the VM was not the best
  • Wanted something which didn't require setup or maintenence
  • Wanted something which I could create once and never have to worry about it again

Cue Docker

What I knew about Docker was...

  • Docker used containers and could spin up an exact instance on any machine with Docker installed.
  • Docker Would allow me to focus on the product, not how to run it
    • Couple weeks back I attended a talk where someone demoed building a simple Docker container
    • This showed me what Docker was capable of

Docker Research

I started with figuring out what components would be needed for my basic lamp stack

  • PHP
    • Composer
  • Apache Web Server
  • MySQL
